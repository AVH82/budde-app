# Terminology — Vocabulaire officiel Budd€

## Règle générale

Employer le vocabulaire du terminal. Budd€ n'est pas une simple application : c'est un terminal personnel opéré par un utilisateur.

## Termes officiels

| Terme | Définition | Usage |
| --- | --- | --- |
| Budd€ | Assistant et identité du projet. | Nom du produit et du personnage. |
| Terminal | Interface globale de Budd€. | Préférer à « app » dans l'univers produit. |
| Opérateur | Personne qui utilise Budd€. | Préférer à « utilisateur » dans les textes immersifs. |
| Module | Vue ou fonction spécialisée du terminal. | Accueil, dépenses, budget, statistiques, commerçants, paramètres. |
| Panneau | Surface encadrée contenant un groupe d'informations. | Cartes, formulaires, états. |
| Cadre | Structure visuelle qui contient le terminal. | Frame system, bords, plaques. |
| Configuration | Réglages du terminal. | Préférer à « settings » dans l'UI française. |
| Archivage | Mise à l'écart contrôlée d'un élément ancien. | Ne pas confondre avec suppression. |
| Réactivation | Retour d'un état dormant vers l'état opérationnel. | Démarrage, réveil, récupération. |
| Trustomètre | Indicateur de confiance ou fiabilité. | Jauge dédiée, pas un simple score générique. |
| Scanner | Module d'analyse visuelle/OCR. | Reçus, justificatifs, lecture locale. |
| Reçu | Document analysé par OCR. | Préférer à « ticket » si l'objet est comptable. |
| Dépense | Sortie financière enregistrée. | Terme métier central. |
| Budget | Enveloppe ou limite de dépense. | Terme métier central. |
| Commerçant | Entité chez qui une dépense est réalisée. | Préférer à « merchant » dans l'UI. |
| Sauvegarde | Copie contrôlée des données. | Locale JSON ou Google Drive appDataFolder. |
| Synchronisation | Échange automatique bidirectionnel. | Ne pas employer si seule une sauvegarde manuelle existe. |

## Termes à éviter progressivement

| Terme à éviter | Remplacement | Raison |
| --- | --- | --- |
| User | Opérateur | Cohérence française et immersive. |
| App | Terminal / Budd€ | Budd€ doit rester un objet logiciel incarné. |
| Screen | Module / vue | Renforce la logique modulaire. |
| Widget | Instrument / panneau / module | Trop générique. |
| Dashboard | Tableau de bord / module d'accueil | À franciser selon contexte. |
| Settings | Configuration / paramètres | Éviter le mélange anglais-français. |
| Sync | Synchronisation | Seulement si techniquement exact. |
| AI buddy | Budd€ | Le personnage a un nom officiel. |

## Règle de nommage éditorial

- Dans les documents techniques, les noms de fichiers, classes et clés peuvent rester en anglais si le code existant l'exige.
- Dans l'interface française, préférer les termes officiels français.
- Ne jamais renommer une notion métier sans mettre à jour ce document.
