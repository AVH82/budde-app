# Budd€ — Design System UI

## Signature visuelle Pip-Boy

Budd€ utilise une signature unique pour les contrôles interactifs afin que l'utilisateur identifie immédiatement l'état d'un bouton, sans modifier la logique métier.

### États OFF

- Couleur dominante : vert foncé, proche du fond terminal.
- Bordure : vert atténué ou doré très discret.
- Rendu : gravé dans l'interface, avec une ombre interne noire.
- Luminosité : réduite afin de ne pas concurrencer le contenu actif.
- Halo : aucun halo externe en état inactif.

### États ACTIF

- Couleur dominante : vert Pip-Boy lumineux.
- Bordure : vert clair `#b9ff55`.
- Rendu : immédiatement identifiable, mais jamais agressif.
- Luminosité : légère augmentation uniquement.
- Halo : diffus, court et contrôlé autour du contrôle actif.

### Halo Pip-Boy

La classe réutilisable `pipboyHalo` porte les bases de la signature active/inactive :

- `pipboyHalo is-off` : contrôle discret, vert sombre, sans halo.
- `pipboyHalo is-active` : contrôle actif, halo vert diffus, luminosité légèrement renforcée.

Les contrôles qui possèdent déjà un état applicatif `.active` doivent reprendre la même logique visuelle que `pipboyHalo is-active`.

### Transitions

- Durée cible : 180 ms.
- Plage autorisée : 150 à 200 ms maximum.
- Propriétés animées : couleur, bordure, halo, luminosité et texte.
- À éviter : animations longues, pulsations agressives, rotations automatiques ou effets distrayants.

### Cohérence visuelle

- La navigation basse, le bouton Paramètres et les boutons principaux doivent partager la même grammaire OFF/ACTIF.
- Un seul élément de navigation doit être visuellement actif à la fois.
- Le bouton Paramètres utilise officiellement le symbole radioactif `☢`.
- Quand la page Paramètres est ouverte, `☢` est allumé ; sinon il reste discret et gravé.
- Les actions principales comme `ANNULER`, `ENREGISTRER`, `AJOUTER` et `VALIDER` peuvent recevoir cette signature seulement si la lisibilité reste prioritaire.

## Sprint 3.0.0 — Terminal UI métal / CRT

La refonte Pip-Terminal transforme l'interface en écran physique rétro-futuriste : l'application doit évoquer un terminal industriel plutôt qu'une page Web.

### Métal et cadres

- Les zones majeures (`header`, cartes, modales, navigation) utilisent un cadre graphite/acier avec relief léger.
- Le métal reste discret : nuances graphite, acier sombre, bords usés et reflets fins.
- Les rivets sont réservés au header Buddy ou aux cadres structurants ; ils ne doivent pas devenir décoratifs à outrance.
- Les grands aplats verts sont proscrits hors états lumineux.

### Écrans CRT

- Chaque carte doit être lue comme un écran encastré dans un cadre métallique.
- Les effets CRT sont subtils : scanlines fines, verre sombre, halo interne contrôlé.
- Le contenu reste prioritaire ; aucune texture ne doit gêner la lecture des montants, champs ou diagnostics.

### Relief et commandes

- Les boutons principaux sont des commandes de terminal : biseau sombre, bord métal, état actif lumineux.
- Les transitions sont limitées à 200 ms maximum.
- Les animations autorisées : variation de luminosité, halo court, changement de bordure ou de filtre.

### Halo et voyants

- Le vert Pip-Boy est réservé aux éléments lumineux : onglet actif, bouton primaire, voyant `☢`, montants importants.
- Le halo doit entourer le pictogramme ou le voyant, pas remplir tout le bouton.
- L'état OFF est gravé, sombre et sans halo externe.
- L'état ON est rétroéclairé avec un halo CRT doux.

### Header Buddy

- Buddy est intégré dans l'écran principal du terminal.
- Sa bulle fait partie du même écran CRT et ne flotte plus sur la page.
- Le header porte le cadre métallique principal avec relief, bord usé et rivets discrets.

### Navigation basse

- Chaque icône est intégrée dans un panneau métallique indépendant.
- Les onglets inactifs ressemblent à du verre sombre encastré.
- L'onglet actif illumine l'icône avec un halo vert contenu.

### Responsive iPhone

- Le header fixe Buddy définit une hauteur dédiée via `--top-h` afin de ne pas masquer le contenu.
- La navigation basse reste fixe et garde ses panneaux métalliques.
- Les variantes mobiles réduisent la hauteur du header et la taille des écrans sans supprimer la hiérarchie terminal.

## Sprint 3.0.2 — Dock Pip-Terminal

Le bandeau inférieur devient un Dock Pip-Terminal permanent, disponible sur tous les écrans sans surcharger le header Buddy.

### Structure du dock

- Le dock est fixe en bas de l'écran et respecte `env(safe-area-inset-bottom)` pour les iPhone.
- Il contient deux lignes distinctes : une ligne d'actions globales, puis une ligne de navigation.
- La hauteur du dock est déclarée via `--nav-h` afin que le contenu principal conserve un padding inférieur suffisant et ne soit jamais recouvert.

### Actions globales

- La première ligne porte uniquement les actions globales `SCANNER` et `AJOUTER`.
- Ces commandes sont accessibles depuis tous les onglets.
- Les boutons utilisent la grammaire Pip-Terminal : métal sombre, relief interne, texte vert rétroéclairé et zone tactile confortable.
- `SCANNER` est légèrement prioritaire visuellement : largeur un peu plus généreuse, bordure plus lumineuse et halo contrôlé.
- Ces actions ne doivent pas porter de logique métier propre dans le dock : elles déclenchent les flux existants.

### Navigation

- La seconde ligne conserve les 5 onglets existants : Home, Dépenses, Budget, Statistiques et Merchants.
- Le style des onglets reste celui du terminal métallique actuel.
- Le halo actif est conservé et un seul onglet de navigation peut être actif à la fois.
- Les écrans métier ne réaffichent pas les raccourcis scanner/ajouter : ces commandes appartiennent au dock.

### Accueil tableau de bord

- L'accueil est le seul écran qui porte le résumé `Budget mensuel` et `Solde disponible`.
- Les gros boutons Scanner/Ajouter ne sont plus dans le tableau de bord : ils sont déplacés dans le dock global.
- Buddy reste permanent dans le header et accompagne le tableau de bord sans être déplacé dans la navigation.

## Sprint 3.3.0 — TRUST Gauge Buddy

La jauge `TRUST` est un composant officiel du bandeau Buddy. Elle exprime uniquement la confiance OCR de Buddy et ne doit pas être réutilisée comme jauge de batterie, budget ou progression.

- Style : monochrome vert Pip-Boy, terminal Vault-Tec, sans rouge/orange.
- Libellés : `TRUST` et échelle `LOW ─ HIGH`.
- Le secteur `HIGH` possède un arc plus épais que le secteur `LOW` pour rendre la confiance élevée immédiatement lisible.
- Animation : l'aiguille revient à une position neutre au début d'une analyse puis rejoint son niveau cible en environ 250 ms ; pas de pulsation ni d'effet spectaculaire.
- Calcul : le score agrège commerçant reconnu, libellé fort (`TOTAL EUR`, `TOTAL TTC`, `NET À PAYER`, `À PAYER`, `MONTANT PAYÉ`), paiement carte (`CARTE BLEUE`, `CB`, `VISA`, `MASTERCARD`, `AMEX`), confirmations multiples, date lisible et cohérence OCR. Les malus couvrent montants contradictoires, OCR pauvre, montant d'article, commerçant inconnu et montants techniques (`TVA`, `DEEE`, remises, fidélité, sous-total).
- Usage : Buddy reste l'unique narrateur. Les explications de confiance apparaissent dans le bandeau supérieur, jamais dans le corps de l'écran hors diagnostic développeur.
